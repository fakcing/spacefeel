Я разрабатываю аниме-раздел для проекта spacefeel на Next.js 14 (App Router) + Tailwind CSS. Мне нужно реализовать продвинутый компонент плеера с двойной системой выбора: Сервер и Озвучка.

1. Источники данных (Backend/Services):
Реализуй или опиши логику трех сервисов:

Yummy Anime (Основной): Возвращает iframe агрегатора.

AniLibria (Качество): Использует официальный API v3 для получения iframe или прямой ссылки.

AnimeVost (Custom): По логике Semolik/AnimeVostPlayer, получает JSON со списком серий и прямыми HLS-ссылками (.m3u8).

2. Компонент плеера (AnimePlayer.tsx):

State Management: Используй useState для хранения activeServer ('yummy', 'libria', 'vost') и activeTranslation (ID озвучки).

Интерфейс выбора (UI):

Верхний ряд: Крупные кнопки переключения серверов: "Yummy (Main)", "AniLibria (HQ)", "AnimeVost (Fast)".

Нижний ряд: Динамический список озвучек/переводов.

Если выбран Yummy — показывай озвучки, которые приходят от их API.

Если AnimeVost — выводи список серий с выбором качества (SD/HD), используя hls.js для воспроизведения.

Адаптивность: На мобилках кнопки выбора должны превращаться в горизонтальный скролл.

3. Техническая реализация:

Для AnimeVost интегрируй кастомную оболочку (можно на базе Plyr или простого видео-тега с hls.js), чтобы плеер выглядел бесшовно.

Реализуй Skeleton Loader: пока данные о серверах подгружаются, показывай красивую заглушку в стиле "Space Dark".

Все запросы должны быть на 'use client', чтобы избежать блокировок со стороны Vercel-серверов.

4. Дизайн (Tailwind):

Сделай акцент на активном сервере с помощью неонового свечения (индиго или фиолетовый).

Плеер должен иметь скругленные углы (rounded-3xl) и быть вписан в контейнер aspect-video.

Напиши код компонентов и логику агрегации данных в lib/playerService.ts
