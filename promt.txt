Я разрабатываю киносайт spacefeel на Next.js (App Router). Для воспроизведения аниме я использую API Yani TV (балансер Kodik).

Проблема: При деплое на Vercel плеер выдает ошибку 404 Not Found для видео-файлов (.mpd), хотя локально на localhost всё работает. Это происходит из-за того, что ссылка на плеер запрашивается на сервере (Vercel IP), а воспроизводится у клиента (User IP), и Kodik блокирует такой запрос из-за несовпадения IP-адресов.

Задача: > 1. Напиши клиентский компонент React ('use client'), который будет принимать tmdbId или shikimoriId в качестве пропсов.
2. Компонент должен выполнять запрос к API Yani TV (https://api.yani.tv/) строго на стороне клиента (через useEffect или useSWR), чтобы запрос уходил с IP-адреса пользователя.
3. Токен API (YANI_TV_TOKEN) должен браться из переменных окружения. Если для клиентского запроса нужен NEXT_PUBLIC_, учти это.
4. Добавь состояние загрузки (Skeletong/Loader) и обработку ошибок (если плеер не найден или заблокирован).
5. Выведи результат в виде защищенного iframe.

Используй Tailwind CSS для стилизации контейнера плеера (соотношение сторон 16:9)
