Ты — Senior FullStack Developer.

Твоя задача — реализовать систему получения видеоплееров через **HTML-парсинг сайтов** для проекта онлайн-кинотеатра **SpaceFeel**.

Проект уже существует. Нужно добавить backend-систему парсинга источников видео.

Технологический стек проекта:

Frontend

* Next.js 14.2 (App Router)
* React 18
* TypeScript
* Tailwind CSS
* Framer Motion

State

* Zustand

Authentication

* NextAuth.js v5

Database

* Prisma ORM
* PostgreSQL (Neon)

Deployment

* Vercel

API источники:

* TMDB API — метаданные (фильмы, сериалы, мультфильмы)
* Yani TV API — аниме
* AniLibria API — аниме

Важно:

VideoCDN API больше **не используется**.

Для **фильмов, сериалов и мультфильмов** необходимо реализовать **парсинг сайтов**, которые содержат встроенные видеоплееры.

---

Цель системы:

Когда пользователь открывает страницу фильма, сериала или мультфильма:

1. Страница получает tmdbId.
2. Backend получает imdbId через TMDB API.
3. Backend запускает несколько парсеров источников.
4. Каждый парсер находит iframe видеоплеера на сайте.
5. Список найденных серверов возвращается frontend.
6. Результат кешируется в базе данных.

---

Необходимо реализовать:

1. API endpoint

/app/api/player/[tmdbId]/route.ts

Endpoint:

GET /api/player/{tmdbId}?type=movie|tv

Ответ должен быть:

{
"servers": [
{
"name": "Server 1",
"iframe": "https://..."
},
{
"name": "Server 2",
"iframe": "https://..."
}
]
}

---

2. Получение imdb_id

Создать helper:

/lib/tmdb.ts

Функция:

getExternalIds(tmdbId, type)

Она должна получать imdb_id через TMDB API.

---

3. Реализовать парсеры сайтов

Создать сервисы:

/services/parsers/source1.ts
/services/parsers/source2.ts
/services/parsers/source3.ts

Каждый сервис должен:

1. принимать imdb_id
2. искать страницу фильма на сайте
3. загружать HTML страницы
4. извлекать iframe плеера
5. возвращать URL плеера

Использовать:

* axios
* cheerio

Если сайт требует JavaScript-рендеринг — использовать Playwright.

---

4. Player aggregator

Создать файл:

/lib/playerService.ts

Он должен:

* запускать все парсеры
* собирать найденные iframe
* фильтровать ошибки
* возвращать список серверов

---

5. Prisma кеш

Добавить модель:

model PlayerCache {
id        Int      @id @default(autoincrement())
tmdbId    Int
type      String
source    String
iframe    String
createdAt DateTime @default(now())

@@unique([tmdbId, source])
}

Логика:

* если кеш младше 24 часов — использовать его
* иначе запускать парсеры заново

---

6. Типизация

Создать тип:

/types/player.ts

export interface PlayerServer {
name: string
iframe: string
}

7. Frontend интеграция

Показать пример страницы:

app/movies/[id]/page.tsx

Страница должна:

1. получать tmdbId
2. запрашивать /api/player
3. отображать iframe
4. давать возможность переключать серверы

UI пример:

Server 1 | Server 2 | Server 3
