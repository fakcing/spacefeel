Ты — Senior Frontend Developer. Тебе нужно реализовать компонент AnimePlayer для проекта spacefeel на Next.js 14 (App Router).

Цель: Создать плеер с возможностью переключения между тремя серверами: Yummy Anime (по умолчанию), AniLibria и AnimeVost.

Технические требования к серверам:

Yummy Anime (Default): Используй стандартную логику вставки через iframe по shikimori_id или tmdb_id. Это основной сервер, который загружается первым.

AniLibria: Реализуй получение данных через их официальный API (https://anilibria.top/api/docs/v1).

Нужно сначала получить данные релиза по shikimori_id.

Извлечь список серий и доступные плееры.

Использовать их iframe или прямой поток (если возможно).

AnimeVost: Реализуй логику на основе репозитория Semolik/AnimeVostPlayer.

Это должен быть кастомный плеер (используй hls.js и plyr-react).

Нужно делать запросы к API AnimeVost (api.animevost.org), получать список серий и прямые ссылки на .m3u8.

Функционал компонента:

Server Selector: Сделай горизонтальный ряд кнопок (Tabs) над плеером для выбора сервера.

Sub-menu (Озвучки/Серии): - Для AniLibria и AnimeVost выведи список серий и доступных озвучек под плеером.

При смене сервера состояние (выбранная серия) должно по возможности сохраняться.

State Management: Используй useState для управления текущим сервером, серией и качеством.

Дизайн: Используй Tailwind CSS. Стиль "Space Dark": темные кнопки с полупрозрачностью (backdrop-blur), активный сервер подсвечивается неоновым цветом (например, #6366f1).

Loading State: Пока данные с API AniLibria или AnimeVost подгружаются, показывай Skeleton-заглушку.

Структура кода:

Вынеси логику запросов к API в отдельные сервисы: services/anilibria.ts и services/animevost.ts.

Используй 'use client', так как плеер требует взаимодействия с браузером и библиотеками hls.js.

Обеспечь полную типизацию на TypeScript.

Напиши полный код компонента и необходимых сервисов.
