# 🔧 Проблемы с озвучками (Anilibria и другие)

## Проблема

Некоторые озвучки работают нестабильно:
- ✅ "Одинокий рокер" (Anilibria) — работает
- ❌ "Токийский Гуль" (Anilibria) — не работает
- ❌ Другие аниме с той же озвучкой — могут не работать

## Причина

### Разные домены плееров

Одна и та же озвучка (например, Anilibria) использует **разные домены** для разных аниме:

```
Одинокий рокер:
→ https://cdn1.anilibria.tv/...
→ ✅ Работает

Токийский Гуль:
→ https://cdn2.anilibria.ru/...
→ ❌ Блокируется (RKN, гео-блокировка, и т.д.)
```

### Почему это происходит

1. **Географические ограничения** — некоторые CDN блокируются по IP
2. **Разные хостинги** — Anilibria использует несколько CDN (cdn1, cdn2, cdn3...)
3. **Блокировки провайдеров** — разные домены имеют разную доступность
4. **Vercel IP** — серверы Vercel могут быть заблокированы

---

## Решение

### 1. **Автоматическая повторная загрузка**
При ошибке плеер автоматически пробует загрузиться до 2 раз:
```
Попытка 1 → Ошибка
Попытка 2 → Ошибка
Попытка 3 → Альтернативный плеер (если есть)
```

### 2. **Кнопка "Сменить озвучку"**
Если текущая озвучка не работает, можно быстро переключиться на другую:
```
[Сменить озвучку] [Попробовать снова (1/2)]
```

### 3. **Индикаторы доступности**
В селекторе озвучек показывается, какие озвучки доступны для текущей серии:
```
[SUB]        ← доступно (белый текст)
[AniLibria] N/A  ← недоступно (серый текст + метка)
[SHIZA]      ← доступно (белый текст)
```

### 4. **Альтернативный плеер**
Если у видео есть `alternative_iframe_url`, плеер автоматически переключится на него после 2 неудачных попыток.

### Типы данных (`types/yani.ts`)
```typescript
export interface YaniVideo {
  // ...
  alternative_iframe_url?: string  // ← Новый fallback URL
  data: {
    alternative_player?: string  // ← Альтернативный плеер
  }
}
```

### Компонент плеера (`components/anime/AniPlayerModal.tsx`)

**Состояния:**
```typescript
const [iframeRetryCount, setIframeRetryCount] = useState(0)
const [useAlternativePlayer, setUseAlternativePlayer] = useState(false)
const MAX_IFRAME_RETRIES = 2
```

**Логика:**
1. Загружаем основной плеер
2. При ошибке → пробуем снова (до 2 раз)
3. После 2 ошибок → переключаемся на альтернативный плеер
4. Если альтернативы нет → показываем кнопку "Сменить озвучку"

**UI обновлений:**
- Счётчик попыток: `"Попробовать снова (1/2)"`
- Кнопка "Сменить озвучку" при таймауте/ошибке
- Индикаторы "N/A" для недоступных озвучек

## Как использовать

### Если озвучка не работает:

1. **Подождите 15 секунд** — сработает таймаут
2. **Нажмите "Попробовать снова"** — плеер перезагрузится
3. **Если не помогло** — нажмите "Сменить озвучку"
4. **Или выберите другую озвучку** вручную из селектора

### Если проблема повторяется:

1. Откройте DevTools → Console
2. Посмотрите ошибку (обычно `ERR_CONNECTION_TIMED_OUT` или `ERR_BLOCKED_BY_CLIENT`)
3. Попробуйте VPN или смените провайдера

### Автоматическое переключение
```typescript
const retryIframeLoad = useCallback(() => {
  if (iframeRetryCount < MAX_IFRAME_RETRIES) {
    // Пробуем снова
    setIframeRetryCount(prev => prev + 1)
    setIframeKey(prev => prev + 1) // Пересоздаём iframe
  } else if (!useAlternativePlayer) {
    // Переключаемся на альтернативный плеер
    setUseAlternativePlayer(true)
  }
}, [iframeRetryCount, useAlternativePlayer])
```
